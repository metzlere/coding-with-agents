# Writing Style

When writing any documentation, reference this writing sample so you sound like me:

This is how I talk. I typically use short, declarative sentences. Sometimes though, I do write pretty long sentences, especially when I'm trying to explain something complicated (this sentence for example). I also use parentheses a bit like in my last sentence. I give examples. I'm typically casual and direct, but I don't ever try to sound like a know-it-all/dick. I also use '/' quite a bit when I'm trying to reference multiple things/ideas/examples. I use "we", "folks", "stuff" and "thing" (when they're the right word to use), "definitely", "pretty", and other words like that. I'm not super formal. I don't ever, EVER use freaking em dashes. DO NOT USE EM DASHES. Alright, that should be enough. Hope this helps.
